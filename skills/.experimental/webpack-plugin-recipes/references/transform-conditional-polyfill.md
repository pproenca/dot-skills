## Inject Polyfills Conditionally Based on Target Browsers

## Problem

You serve two bundles: `legacy.js` (ES5 + full polyfill set for IE11/older Safari, served via `<script nomodule>`) and `modern.js` (ES2022 + minimal polyfill for last-2-versions browsers, served via `<script type="module">`). Manually maintaining `import 'core-js/stable'` at the entry — and the conditional `import 'array-flat-polyfill'` next to it — couples your source code to your target browser list. Tomorrow when you drop IE11 support, you have to find every polyfill import and remove the obsolete ones.

You want the plugin to read your `.browserslistrc`, decide which features are NOT supported by the target browsers, and inject ONLY those polyfills into the entry. The result: modern bundle ships ~5kb of polyfills; legacy bundle ships ~40kb; no source code changes needed.

## Pattern

In `apply()`, compute the needed polyfill list once (using `browserslist` + `core-js-compat`). Tap `compiler.hooks.entryOption` (or use `EntryPlugin` directly) to prepend a generated polyfill module to each entry. Emit the polyfill module as a virtual module containing only the side-effect imports for needed features.

**Incorrect (without a plugin — hand-maintained import at entry):**

```js
// src/index.ts — entry
import 'core-js/stable';                  // ships ~40kb in BOTH modern and legacy bundles
import 'regenerator-runtime/runtime';     // unnecessary in modern (async/await native)
import './app';

// Modern bundle: still 40kb of polyfills no modern browser needs.
// Switch to dropping IE11? Forget to remove the imports → still 40kb.
```

**Correct (with this plugin — auto-injected from browserslist):**

```js
const { validate } = require('schema-utils');

const schema = {
  type: 'object',
  properties: {
    targets: {
      oneOf: [
        { type: 'string', description: 'browserslist query, e.g. "last 2 versions"' },
        { type: 'array', items: { type: 'string' } },
        { type: 'object' },
      ],
      description: 'Override .browserslistrc (defaults to the user\'s config)',
    },
    entries: {
      type: 'array',
      items: { type: 'string' },
      description: 'Restrict to these entry names (default: all)',
    },
    coreJsVersion: { type: 'string' },
    modules: { enum: ['core-js', 'core-js-bundle'] },
  },
  additionalProperties: false,
};

const DEFAULTS = { coreJsVersion: '3.36', modules: 'core-js' };

class ConditionalPolyfillPlugin {
  constructor(options = {}) {
    validate(schema, options, { name: 'ConditionalPolyfillPlugin', baseDataPath: 'options' });
    this.options = { ...DEFAULTS, ...options };
  }

  apply(compiler) {
    const PLUGIN = 'ConditionalPolyfillPlugin';
    const { EntryPlugin } = compiler.webpack;

    const polyfills = this.computeNeededPolyfills();
    if (polyfills.length === 0) return; // modern targets need nothing

    // Build the polyfill module source
    const polyfillSource =
      `// Auto-generated by ConditionalPolyfillPlugin\n` +
      `// Targets: ${this.targetsLabel()}\n` +
      polyfills.map((p) => `import ${JSON.stringify(p)};`).join('\n');

    const virtualPath = path.join(compiler.context, '__polyfills__', 'index.js');

    // 1) Provide the source via virtual-module mechanism (see virtual-module-from-memory)
    compiler.hooks.compilation.tap(PLUGIN, (compilation) => {
      const NormalModule = compiler.webpack.NormalModule;
      NormalModule.getCompilationHooks(compilation).readResource
        .for(undefined)
        .tap(PLUGIN, (loaderContext) => {
          if (loaderContext.resourcePath !== virtualPath) return;
          return Buffer.from(polyfillSource, 'utf8');
        });
    });

    // 2) Inject as an additional entry point — runs BEFORE the main entry
    const entryNames = this.options.entries
      ?? Object.keys(compiler.options.entry ?? {});

    for (const name of entryNames) {
      new EntryPlugin(compiler.context, virtualPath, { name }).apply(compiler);
    }
  }

  computeNeededPolyfills() {
    // Cheap version using core-js-compat
    try {
      const compat = require('core-js-compat');
      const browserslist = require('browserslist');
      const targets = this.options.targets
        ? browserslist(this.options.targets)
        : browserslist();

      const { list } = compat({
        targets,
        version: this.options.coreJsVersion,
        modules: this.options.modules === 'core-js' ? ['es', 'web'] : undefined,
      });
      return list.map((m) => `${this.options.modules}/modules/${m}.js`);
    } catch (e) {
      // core-js-compat not installed — leave a clear hint
      throw new Error(
        `ConditionalPolyfillPlugin: requires 'core-js-compat' and 'browserslist' ` +
        `as devDependencies. Install: npm i -D core-js-compat browserslist`,
      );
    }
  }

  targetsLabel() {
    if (Array.isArray(this.options.targets)) return this.options.targets.join(', ');
    if (typeof this.options.targets === 'string') return this.options.targets;
    return '(browserslist default)';
  }
}

module.exports = ConditionalPolyfillPlugin;
```

## Usage

```js
// Legacy bundle config
{
  target: 'browserslist:legacy',
  plugins: [new ConditionalPolyfillPlugin({ targets: 'IE 11, last 5 versions' })],
}

// Modern bundle config
{
  target: 'browserslist:modern',
  plugins: [new ConditionalPolyfillPlugin({ targets: 'last 2 chrome versions, last 2 firefox versions' })],
}

// package.json
{
  "browserslist": {
    "legacy": ["IE 11", "last 2 versions"],
    "modern": ["last 2 chrome versions", "last 2 firefox versions"]
  }
}
```

## How it works

- **`EntryPlugin`** is webpack's official way to add an entry point at apply time (replaces the old `compiler.options.entry` mutation). Adding the polyfill as a separate entry makes it run BEFORE the main entry's code.
- **Virtual module** delivers the polyfill module without writing to disk — see [`virtual-module-from-memory`] for the underlying pattern
- **`core-js-compat`** is the canonical "what does my browserslist need?" computation; reused by `@babel/preset-env` so the polyfill list matches what Babel would inject
- **One entry, multiple polyfill sets** — each call to `EntryPlugin` adds a separate entry; webpack deduplicates the virtual module across them
- **Throwing if `core-js-compat` not installed** — the plugin can't function without it, so failing at construction is better than silent no-op

## Variations

- **Per-feature polyfill (not core-js)**: for `IntersectionObserver`, `ResizeObserver` — use [polyfill-library](https://github.com/Financial-Times/polyfill-library)
- **Differential serving** (auto-emit BOTH modern and legacy entries): instantiate the plugin twice with different targets
- **Skip polyfills for modules that opt out** (a `/* no-polyfill */` comment in source): combine with a loader that flags entries
- **Polyfill only on specific imports** (e.g., only polyfill the `Intl` chunk): create polyfill entry per chunk

## When NOT to use this pattern

- You use `@babel/preset-env` with `useBuiltIns: 'usage'` — Babel injects per-file polyfill imports, often a better fit
- You target only modern browsers and need zero polyfills — skip this plugin
- You use SWC's `env` target — SWC handles polyfill injection itself
- You're already using `Polyfill.io` (delivers polyfills via CDN based on User-Agent) — different model entirely
