## Trim the test-build bundle for the Vitest addon

When `@storybook/addon-vitest` runs your play functions, it spins up a stripped Storybook bundle. By default that bundle still includes docs blocks, MDX entries, auto-generated docs pages, and addon panels that play functions never use. The `main.ts` `build.test` config block tells Storybook to skip those during the *test* build only — the regular `storybook dev` and `storybook build` outputs stay full-featured. This cuts the per-test bundle size, reduces Vitest startup, and removes a class of "addon broke my play function" failure mode.

> **Storybook 9+ note:** On-demand story bundling (sidebar populated from index, story modules compiled lazily on click) is now the *default* behavior for the dev server — no main.ts knob required. The `build.test.*` keys below are only for the test build.

**Incorrect (default test bundle — includes docs/MDX/a11y panels that play functions don't use):**

```ts
// .storybook/main.ts
const config = {
  framework: '@storybook/react-vite',
  stories: ['../src/**/*.stories.@(ts|tsx)'],
  addons: ['@storybook/addon-docs', '@storybook/addon-a11y', '@storybook/addon-vitest'],
  // No build.test config — Vitest runs against the full Storybook bundle
} satisfies StorybookConfig;
```

**Correct (test build trims everything play functions don't touch):**

```ts
// .storybook/main.ts
const config = {
  framework: '@storybook/react-vite',
  stories: ['../src/**/*.stories.@(ts|tsx)'],
  addons: ['@storybook/addon-docs', '@storybook/addon-a11y', '@storybook/addon-vitest'],

  build: {
    test: {
      // Addons that don't contribute to running play functions
      disabledAddons: ['@storybook/addon-docs'],
      // Skip MDX-only entries — they have no play function to run
      disableMDXEntries: true,
      // Skip generated Docs pages — same reason
      disableAutoDocs: true,
      // Skip the docs-blocks bundle (saves ~200kb)
      disableBlocks: true,
    },
  },
} satisfies StorybookConfig;
```

**Reference table:**

| `build.test` key | Effect |
|------------------|--------|
| `disabledAddons` | Skip listed addons during test build |
| `disableMDXEntries` | Skip MDX files (no play function = nothing to test) |
| `disableAutoDocs` | Skip Docs pages generated by `tags: ['autodocs']` |
| `disableBlocks` | Skip the doc-blocks bundle (`Canvas`, `ColorPalette`, etc.) |

**Profiling cold start:**

```bash
storybook dev --loglevel debug    # verbose timings for boot phases
storybook build --stats-json      # writes a Vite stats file for bundle analysis
```

**When NOT to use this pattern:**
- Projects without the Vitest addon — the `build.test` block has no consumer; it's a no-op.

**Why this matters:** A trimmed test bundle reduces Vitest setup time (less to evaluate) and removes failure modes from addons that play functions don't exercise. For a library with 300+ stories, this typically saves 30-60s on a cold CI run.

Reference: [Storybook main.config: build](https://storybook.js.org/docs/api/main-config/main-config-build), [Vitest addon](https://storybook.js.org/docs/writing-tests/integrations/vitest-addon)
