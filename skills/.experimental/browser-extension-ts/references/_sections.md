# Rule Categories

## 1. Manifest V3 (mv)
**Impact:** CRITICAL
**Description:** Patterns and best practices for Manifest V3 migration and compatibility. Essential for modern browser extension development.

## 2. Code Style (style)
**Impact:** HIGH
**Description:** Organization, naming conventions, and file structure patterns for TypeScript browser extensions.

## 3. Component Patterns (comp)
**Impact:** HIGH
**Description:** Patterns for structuring UI components, background managers, content script modules, and cross-context communication.

## 4. Error Handling (err)
**Impact:** HIGH
**Description:** Error handling strategies for browser extension contexts including storage failures, URL parsing, and async coordination.

## 5. Testing (test)
**Impact:** MEDIUM
**Description:** Testing patterns, file organization, and strategies for browser extension code.
