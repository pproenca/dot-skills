# Sections

This file defines all sections, their ordering, impact levels, and descriptions.
The section ID (in parentheses) is the filename prefix used to group rules.

---

## 1. Design Foundations (found)

**Impact:** CRITICAL
**Description:** Core design principles including color, typography, spacing, dark mode, and iconography that form the visual language of iOS apps. Getting these right is essential for platform consistency.

## 2. Layout & Spacing (layout)

**Impact:** CRITICAL
**Description:** Safe areas, margins, the 8pt grid system, and responsive layout patterns that ensure content displays correctly across all iOS devices and orientations.

## 3. Navigation Patterns (nav)

**Impact:** HIGH
**Description:** Tab bars, navigation bars, hierarchical navigation, and search patterns that define how users move through your app.

## 4. UI Components (comp)

**Impact:** HIGH
**Description:** Standard iOS components including buttons, text fields, lists, sheets, and alerts with their correct usage patterns and styling.

## 5. Interaction Design (inter)

**Impact:** HIGH
**Description:** Gestures, touch targets, haptic feedback, and input handling that create the tactile experience users expect from iOS apps.

## 6. User Feedback (feed)

**Impact:** MEDIUM-HIGH
**Description:** Loading states, progress indicators, notifications, and error handling patterns that keep users informed about app state.

## 8. Accessibility (a11y)

**Impact:** HIGH
**Description:** VoiceOver support, Dynamic Type, contrast ratios, and inclusive design practices required for accessible iOS apps.

## 7. UX Patterns (ux)

**Impact:** MEDIUM
**Description:** Onboarding, modality, data entry, permissions, and other common user experience patterns with platform-appropriate implementations.
